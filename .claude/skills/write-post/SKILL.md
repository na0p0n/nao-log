# write-post — 日本語技術記事の下書きを生成する

## いつ使うか

- 「記事を書いて」「下書きを作って」「〇〇について投稿して」
- `trend-scan` の結果から記事化するとき
- Issue ラベル `type:post` の対応として新規記事を起こすとき

## 手順

### 1. トピック確認

次の 3 点をユーザーと合意（不明なら聞く）:

- **タイトル案**（30〜60 字）
- **ターゲット読者**（例: 「Astro を触り始めた人」）
- **記事の目的**（入門 / 比較 / 体験記 / 速報 / チュートリアル）

### 2. スラッグを決める

- 英数字とハイフンのみ、全小文字
- 例: `claude-code-hooks-primer`, `astro5-content-layer-intro`

### 3. 章立てを提案

- `##` と `###` のみ使用
- 構成をテキストで提示してユーザー承認を取ってから本文へ
- 典型構成: 導入 → 前提知識 → 本論（手順 or 比較）→ つまずきポイント → まとめ

### 3.5 SEO 下準備

本文を書く前に次を決め、ユーザーに提示して合意を取る:

- **一次キーワード** 1 個（記事の検索流入の主軸）
- **副次キーワード** 2〜3 個（関連語・類似表現）
- **タイトル候補 3 案**
  - 30〜60 字
  - 一次キーワードは先頭 1/3 以内
  - 数字・具体名詞・年号で CTR を意識（例: 「3 つの観点」「2026 年版」）
- **description 候補 2 案**（80〜120 字、「読むと何が得られるか」を書く）

内部リンクが作れそうか、`Grep` で `src/content/posts/` を走査して関連既存記事を確認。

### 4. 本文を執筆（ルール）

- 日本語、**ですます調**で統一
- 一文は 80 字前後を目安（100 字を超えたら分ける）
- コードブロックには必ず言語指定（` ```ts `, ` ```bash ` など）
- 半角英数と日本語の間は半角スペース
- 画像は `public/assets/<slug>/` に配置し、相対パスで参照。`alt` 必須
- 事実情報には一次情報の URL を脚注か本文リンクで示す
- **SEO 配置**:
  - 一次キーワードは **導入 1 段落目** と **結論** に必ず登場
  - `##` の少なくとも 1 つに一次キーワードか関連語を含める
  - 副次キーワードは各 `##` に 1 つずつ散らす
  - 内部リンク（3.5 で洗い出した関連記事）を本文中に自然に挿入

### 5. 保存

`src/content/posts/<slug>.md` に以下の frontmatter で書き出す:

```yaml
---
title: "（記事タイトル）"
description: "（80〜120 字の要約。検索結果に出る想定）"
pubDate: YYYY-MM-DD
tags: []
draft: true
# heroImage: "/assets/<slug>/og.png"  # OG image を用意したら有効化
---
```

本文末尾に必ず次の行を入れる:

```
---

本記事は Claude の支援を受けて作成しています。
```

### 6. レビュー依頼

保存後、ユーザーに次を提案:

> `post-reviewer` サブエージェントに文体・構成レビューを依頼しますか？
> 事実確認は `fact-checker` で別途走らせられます。

## やらないこと

- 事実確認なしでの数値・仕様・バージョン・価格の記載（必ず一次情報にあたる）
- `draft: false` への変更（ユーザーの明示指示がない限り）
- 画像の自動生成・バイナリの作成
- 既存記事の上書き（同名スラッグがあれば確認）
